Hướng dẫn sử dụng

1. Import câu hỏi

Để thêm câu hỏi vào hệ thống:

  • Giao diện web: Vào Quản lý > Import, paste JSON vào textarea và nhấn Import.

Nếu chủ đề đã tồn tại, câu hỏi mới sẽ được thêm vào chủ đề đó.

2. Tạo câu hỏi bằng AI

Chọn loại prompt phù hợp, copy và paste vào ChatGPT / Claude / Gemini. Nhớ sửa chủ đề, số lượng và các yêu cầu cho phù hợp trước khi gửi.

Tạo bộ câu hỏi trắc nghiệm để ôn tập/luyện thi.

Chủ đề: [CHỦ ĐỀ CỦA BẠN]
Số lượng câu hỏi: [SỐ LƯỢNG]
Ngôn ngữ: Tiếng Việt

Yêu cầu:
- Mỗi câu hỏi có đúng 4 đáp án (A, B, C, D)
- Phần lớn câu có 1 đáp án đúng, một số câu có thể
  có nhiều đáp án đúng (đánh dấu tất cả is_correct: true)
- Đa dạng độ khó (dễ, trung bình, khó)
- Bao phủ nhiều khía cạnh của chủ đề
- Chỉ thêm "explanation" khi đáp án không hiển nhiên

Chỉ trả về JSON hợp lệ, không giải thích:

{
  "subject": "[Tên chủ đề]",
  "questions": [
    {
      "content": "Nội dung câu hỏi?",
      "explanation": "Chỉ khi cần",
      "answers": [
        {"label": "A", "content": "...", "is_correct": false},
        {"label": "B", "content": "...", "is_correct": true},
        {"label": "C", "content": "...", "is_correct": false},
        {"label": "D", "content": "...", "is_correct": false}

Tạo bộ câu hỏi trắc nghiệm để ôn tập/luyện thi.

Chủ đề: [CHỦ ĐỀ]

Yêu cầu:
- Mỗi câu hỏi có đúng 4 đáp án (A, B, C, D)
- Đa dạng độ khó (dễ, trung bình, khó)
- Thêm "explanation" khi đáp án không hiển nhiên
- Nội dung hỗ trợ Markdown. Quy tắc format:
  + Xuống dòng trong JSON string dùng \n
  + Code block PHẢI dùng TRIPLE backtick (```),
    KHÔNG dùng single backtick.
    Ví dụ: \n```java\ncode here\n```\n
  + Nếu đáp án là giá trị code ngắn: dùng
    single backtick cho inline code: "`value`"
  + Sơ đồ có cấu trúc (flowchart, UML, ER,
    sequence diagram): dùng ```mermaid block
  + Minh hoạ khác (stack, bộ nhớ, truth table,
    cây nhị phân): dùng ASCII art trong ``` block

Sau khi nhận JSON, paste vào trang Import hoặc lưu file rồi chạy ./quiz import file.json.

3. Chế độ luyện tập

Flashcard — Trả lời từng câu một, xem kết quả ngay. Câu hỏi được xáo trộn và không lặp lại trong cùng phiên.

Thi thử — Chọn số câu, trả lời tất cả rồi nộp bài. Xem điểm và review chi tiết từng câu.

4. Loại câu hỏi

Một đáp án: Hiển thị dạng radio button — chọn 1 đáp án duy nhất.

Nhiều đáp án: Hiển thị dạng checkbox kèm ghi chú "(Chọn nhiều đáp án)" — phải chọn đúng tất cả đáp án đúng mới được tính điểm.

Rich content: Câu hỏi và đáp án hỗ trợ Markdown — code blocks, in đậm/nghiêng, hình ảnh base64.

5. Thống kê

Vào Thống kê để xem tỷ lệ chính xác, điểm trung bình, điểm cao nhất và lịch sử làm bài theo từng chủ đề.
